# Сжатый лог работы над AUV (проект golfeno/auvs)

Краткая хронология задач и решений. Управление — модуль `src/my_auv_control/auv_nav/` (автопилот `autopilot`), модель в `src/my_auv_description/models/submarine.sdf`, мир `src/my_auv_bringup/worlds/static_world.sdf`.

## Модель / рули
- Склонирован репозиторий, переименован воркспейс в `auv`.
- Добавлен **верхний вертикальный руль** (`vertical_rudder_top`, кормовой, выше оси) — стабилизация крена; позже сделан и помощником поворота (курсовой терм + креновой), т.к. иначе гасил поворот.
- Стабилизация крена: все 4 горизонтальных руля (дифференциально) + верхний верт. руль (отдельный канал).
- Раскраска деталей по видам (корпус синий, винты жёлтые, гориз.рули зелёные, верт.рули оранжевые, балласты красные) — через ambient+diffuse+specular.
- 6 рулей интегрированы во все алгоритмы автопилота и миксеры; топики в bridge.

## Баки (балласт)
- 4 бака сделаны одинаковой длины (0.375 м) по всему килю.
- Перебалансировка масс (твой коммит «6.1 рулей»): корпус 122.7, баки ~16.6 → 139.28 кг; затем догрузка +0.7 кг (≈139.98).

## Калибровка массы / нейтрали
- `mass_calibration.py` параметризован по модели; исправлен расчёт Vz (МНК по «свободному окну», отсечка зажима).
- Создан `ballast_neutral_finder.py` (entry: `ballast_neutral`): поиск нейтральной плавучести.
  - v1 бисекция → v2 плавный свип (шаг падает после овершута) → v3 метрика «ускорение» (оказалась ложной: a~0 на любой устойч. скорости) → **v4 метрика = терминальная скорость Vz→0** + settle 20–30с → v5 интерполяция корня + **2-я стадия: регулировка дифферента** (нос/корма trim при сохранении объёма, метрика = угол тангажа asin(2(wy−zx))).
  - Результат: `bz_neutral ≈ 0.493`, дифферент trim ≈ +0.05 (нос баки 1,2 = base+trim; корма 3,4 = base−trim). Вписано в код.

## Критические баги, найденные и исправленные
1. **Миксеры зависали без одометрии** — `return` до чтения клавиш; ручное управление расцеплено с одометрией.
2. **IMU в повёрнутой раме body (+90° по Y)** — `yaw_d` брался как `roll_d` → руль курса рулил не туда (орбиты, низкая V, нет фазы 2). Ремап осей IMU в мировую: roll=imu_yaw, pitch=imu_pitch, yaw=−imu_roll.
3. **Знак глубины** в `depth_rudder.py` — я ошибочно «исправил», аппарат пошёл вниз вместо вверх; откатил к оригиналу `th=+(Kp·z_err+…)`.
4. **Балласт: тег `<topic>` ИГНОРИРУЕТСЯ** в gz-sim8 `BuoyancyEngine` (нет такого параметра в Configure!). Слушает только `/model/<namespace>/buoyancy_engine`. Перешёл на `<namespace>sub_ballast_N` → топики `/model/sub_ballast_N/buoyancy_engine`. Это и было «балласт не влияет».
5. **`max_inflation_rate` по умолчанию 0.000003 м³/с** — бак наполнялся 83 мин. Поставил 0.001 (бак 15л за 15с, реалистично по статьям ~15с полная смена плавучести).
6. **Балласт «взлетал»**: сила на краях ±294 Н = ±2.1 м/с². Ограничен авторитет (`bz_authority=0.07` → ±0.3 м/с²) + slew объёма.
7. **Осцилляция глубины**: переписан регулятор балласта на **каскад с ограничением скорости** — внеш. контур z_err→желаемая Vz (лимит 0.12 м/с, трение гасит), внутр. контур гонит факт. Vz к желаемой. Плавный выход, перелёт ~7 см, без пилы.

## Фазы автопилота (4-фазный автомат)
- **1 NAV** круиз; **2 Z_STAB / Z_CORRIDOR** глубина (2 вариации); **3 APPROACH** стоп→разворот на месте→газ прямо; **4 HOVER_STAB**.
- Радиус сближения `r_app=3 м`; раньше дедлок из-за `dist<0.5`.
- Разворот в фазе 3: полный дифференциал движков + PD (D по гироскопу гасит осцилляцию) + гистерезис 10/20°.
- Вход в фазу 2 зависит от режима: **Z_CORRIDOR только для рулей** (нужен поток); **балласт** входит в Z_STAB строго над/под точкой.
- Жёстче курс (меньше дуг): `turn_first` 40°→20°, `yd_frac` 0.6→0.9, taper cos², gain курса 6; тормоз остаточного хода при развороте на месте.

## Сенсоры
- Подключён IMU; слияние IMU+одометрия+магнитометр через **фильтр Калмана** (1D на ось: predict по гироскопу, update по одометрии; yaw корректируется магнитометром).
- Добавлены **магнитометр, альтиметр (эхолот вниз), сонар (эхолот вперёд)**; в телеметрию выведены IMU/ODO/KALMAN, Mag, Дно, Сонар, углы рулей.
- Сонар: исправлена посадка в нос (рамка body повёрнута), сделан **многолучевым 16×16 (±20° гор × ±20° верт)** для оценки габаритов; bridge PointCloud2 `/sonar/points`, дисплей RViz `SonarCloud`.
- (Замечание: одиночный 1D-веер габариты не даёт — поэтому 2D-сетка лучей.)
